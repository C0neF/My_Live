# 重设计方案 B —— 跟随系统（Follow-system）

> 不给 MyLive 设定固定的明暗「情绪」，而是让它**适配用户环境**：浅色与深色两套同等打磨、互为一等公民，由系统设置驱动。
> 适用判断：更保守、更「实用」的读法——白天通勤亮屏、夜里弱光，应用跟随用户当前环境，而非强加氛围。
> 主旨同样是**让位给画面**，但底色不追求「影院近黑」，而是两种环境下都中性、干净、稳定。

本文件是设计规格（spec），不直接改 app 代码。落地时按「文件映射」逐处替换。

---

## 1. 配色系统（OKLCH 为准，hex 为约值需微调）

关键差异：**两套灰阶必须同等调校**，accent 在两种底上都要过对比度。不像方案 A 把浅色当副模式。

### 浅色

| token | OKLCH | ≈hex | 用途 |
|---|---|---|---|
| `bg` | `oklch(0.985 0 0)` | `#FAFAFB` | 根背景（**非纯白**，让白色封面能分离） |
| `surface` | `oklch(0.96 0 0)` | `#F1F2F4` | 输入框、sheet、抬升块 |
| `hairline` | `黑 10%` | `rgba(0,0,0,.10)` | 仅分隔处 1px 线 |
| `ink` | `oklch(0.23 0.004 250)` | `#1A1C1F` | 主文字 |
| `muted` | `oklch(0.50 0.008 250)` | `#5A5F66` | 次文字、图标 |
| `accent` | `oklch(0.55 0.13 245)` | `#2A6FCB` | 选中态、主操作 |

### 深色

| token | OKLCH | ≈hex | 用途 |
|---|---|---|---|
| `bg` | `oklch(0.19 0.004 250)` | `#121316` | 根背景（比方案 A **略亮**，不做影院近黑） |
| `surface` | `oklch(0.24 0.005 250)` | `#1C1E22` | 输入框、sheet、抬升块 |
| `hairline` | `白 10%` | `rgba(255,255,255,.10)` | 仅分隔处 |
| `ink` | `oklch(0.95 0.003 250)` | `#ECEDEF` | 主文字 |
| `muted` | `oklch(0.70 0.008 250)` | `#9BA0A7` | 次文字、图标 |
| `accent` | `oklch(0.71 0.12 245)` | `#5B9BEA` | 选中态、主操作 |

### 平台色（保留原值，不动）
斗鱼 `#FF5D23` · 虎牙 `#FFD736` · 哔哩哔哩 `#F07775`（`PlatformColors.kt`）。只作小块纯色标记，禁止渐变。

### 对比度校验（两套都要过）
- 浅色：`ink` on `bg` ≈ 14:1 ✓；`muted` on `bg` ≈ 5.2:1 ✓；按钮 `accent`(L0.55)+白字 ≈ 4.6:1 ✓。
- 深色：`ink` on `bg` ≈ 14:1 ✓；`muted` on `bg` ≈ 6:1 ✓；深色按钮用 `accent` 打底配**近黑字**（白字在 L0.71 上不够），或下沉到 L≈0.56 配白字。
- 聊天 / 弹幕走 `ink`，发送者名 `muted` 或单一 `accent`，禁止彩虹名。

### 文件映射
- `ui/theme/Color.kt`：浅色、深色两组 `md_theme_*` 都按上表替换，**两组同等用心**。
- `ui/theme/Theme.kt`：
  - **保留** `isSystemInDarkTheme()` 作为驱动（`Theme.kt:46`），这是本方案的立场。
  - 删掉 `primaryContainer = seedColor.copy(alpha = …)` 半透明 hack（`Theme.kt:57-65`），改用实色 `surface`。

## 2. 结构性改造（A / B 两方案共享，布局前后对比见 `layout-changes.md`）

### 2.1 LiveRoomCard（`component/LiveRoomCard.kt` 整文件）
- 去掉 `Card` 容器、1px 描边、`surfaceVariant` 填色（`LiveRoomCard.kt:51-54`）。
- 结构：圆角封面 `12dp` → `8dp` 间距 → 裸底两行信息。
- **跟随系统下的唯一额外照顾**：浅色模式里裸图直接贴 `#FAFAFB` 容易和白色封面糊在一起 —— 用 `#FAFAFB`（非纯白）做底就够分离；**仍不需要给卡加边框**。深色模式同方案 A，完全裸卡。
- 平台徽章：改纯色小 chip（去 `horizontalGradient`，`:98-106`）。
- 删头像「premium border」（`:157`）;信息块改 YouTube 式:标题占满宽、可 2 行（`ink`）;头像缩到 18–20dp（无环）移到次行与主播名（`muted`）同排。

### 2.2 首页（`screen/home/HomeScreen.kt`）
- wordmark 固定为 `ink`，删随平台变色动画（`:70-77`）。
- 搜索改 `surface` 实底输入区，去描边（`:119-127`）。
- 平台选择器（`:184-221`）改可滚动 tab 行 / 分段控件，选中用平台品牌色下划线或底色。

### 2.3 我的 / 设置 去卡片化（`screen/mine/MineScreen.kt`）
- 干掉 5 张堆叠描边卡（`:104-127`），换分组列表（图标 + 标题 + chevron + `hairline` 分隔），复用 `SettingsMenu` 行样式。
- profile 卡（`:53-100`）：无真账户，建议删或缩成一行；**删营销文案**「极致、纯粹的直播体验」；圆角 `24dp → 12dp`。

### 2.4 空 / 错误状态
- `EmptyState.kt:26-28`：emoji `📭` 换图标集真图标，统一图标语言。
- 文案补具体动作；错误态保留重试。

### 2.5 房间页（`screen/room/LiveRoomScreen.kt`，布局不动）
- 聊天 ≥4.5:1；Super Chat 用平台/身份色**纯色块**，禁止 side-stripe 色条。
- 控件自动隐藏干净；横屏叠层聊天压暗保证可读。

### 2.6 动效
- 150–250ms、ease-out、只表达状态。
- 保留底栏滚动隐藏、Cupertino 转场；删 wordmark 变色动画。

## 3. B 方案的独有取舍

| 维度 | B：跟随系统 |
|---|---|
| 身份 | 不设固定情绪，明 / 暗双模式同为一等公民 |
| 底色 | 浅 `#FAFAFB`、深 `#121316`（比 A 略亮，非影院近黑） |
| 卡片分隔 | 深色裸卡无边框；浅色靠 `#FAFAFB`（非纯白）+ 间距分离 |
| 氛围 | 中性、环境适配、稳定，贴近系统原生观感 |
| 风险 | **两套都要单独验对比度**，工作量约为 A 的两倍；任何只调一套的偷懒都会露馅 |

---

## 4. 落地顺序（建议）
1. `Color.kt` + `Theme.kt`：**两套** token 同时铺好，保留系统驱动，去半透明 hack。
2. `LiveRoomCard.kt`：去卡片化（注意浅色底用 `#FAFAFB` 非纯白）。
3. `HomeScreen.kt`：固定 wordmark + 平台 tab 选择器。
4. `MineScreen.kt`：分组列表替换卡片堆。
5. `EmptyState.kt` + Super Chat：图标 / 纯色块收尾。

---

## A vs B 一句话取舍
- 想要**有记忆点、沉浸、内容最跳** → 选 A（深色优先）。
- 想要**安静、不打扰、跟着用户环境走、最贴合「实用」品牌词** → 选 B（跟随系统），代价是双模式都得认真调。
